# Nader George Academy UI Critique

Target: frontend/src
Total score: 12/40

Main verdict: the interface has a real premium learning-product ambition, but current implementation is blocked by WebGL decorative dependencies that can crash public pages, and the visual system is inconsistent across landing, auth, student, admin, video, and community surfaces.

Priority issues:
- P0: WebGL/RippleGrid/CircularGallery failure crashes /, /login, and /register in Playwright headless with "This page couldn't load". Add non-WebGL fallback and error boundary, and never let decorative canvas own page viability.
- P1: Product identity is split. PRODUCT.md says Cairo and gold warm identity; DESIGN.md says Manrope; code uses Cairo but many components use admin tokens, Facebook blue, generic Tailwind gray/red/emerald, pure white/black, and heavy borders.
- P1: Student UI borrows admin chrome and admin tokens. Student context is mobile-first focused study, but shell structure, icon sidebar, settings, theme controls, decorative panels, and dense cards feel like a management dashboard.
- P1: Registration has too much cognitive load for a teen on mobile: avatars, preview panels, many fields, guardian details, birthday computations, WhatsApp checks, and instructions modal compete with the core goal.
- P2: The design system rulebook is not enforced. There are hundreds of borders despite the no-line rule, default glassmorphism/orbs, hero metric patterns, repetitive cards, and hard-coded colors.

What works:
- Arabic-first RTL is real, not an afterthought.
- Progress, locking, exams, packages, quick access, and resume points match the product's controlled-learning promise.
- Shared tokens and components exist, so cleanup can be systematic rather than a full restart.

Recommended sequence:
1. harden public/auth WebGL fallback.
2. normalize tokens and remove hard-coded colors/borders.
3. distill student dashboard and registration.
4. adapt mobile navigation and touch workflows.
5. polish final states and empty/error states.
