## Resolved Priority Issues

- Operational request failures no longer appear as empty moderation or dashboard queues.
- Clickable rows and containers were replaced or supplemented with semantic keyboard-accessible controls.
- Shared dialogs now trap focus, restore focus, lock background interaction, and expose accessible labels.
- Student dashboard hierarchy prioritizes continuation and the nearest exam while secondary destinations are compact or disclosed.
- Reports no longer combine server and client pagination, and filters apply explicitly.
- Moderation supports guarded bulk actions, partial-failure feedback, and rejection confirmation/reason where supported.
- Landing content is visible by default; decorative motion, particles, and automatic gallery movement were reduced.
- Tiny static labels, fake links, internal English copy, unrelated footer text, and undersized touch targets were addressed.
- Assistant access to permitted admin tools is labeled as an assistant workspace rather than silently crossing boundaries.
