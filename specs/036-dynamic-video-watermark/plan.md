## Summary

Implement a dynamic, randomized UI watermark displaying the student's name and phone number on top of the secure video player (`embed` iframe). The identity data will be securely passed by injecting it into the AES-encrypted video session token generated by the `.NET` backend.

## Technical Context

**Language/Version**: TypeScript (Next.js 16.2.1 / React 19) + C# (.NET 9)  
**Primary Dependencies**: Native DOM elements (MutationObserver, CSS absolute positioning)  
**Storage**: PostgreSQL (for retrieving current student info during session creation)  
**Testing**: Local verification in browser across YouTube / Telegram providers  
**Target Platform**: Web Browsers (Mobile and Desktop)  
**Project Type**: Next.js App Router (Server-side API handlers)  
**Performance Goals**: Zero impact on video playback frame rate.  
**Constraints**: The watermark must not block pointer events (clicks) so UI remains usable.  
**Scale/Scope**: Displayed on all videos for all students globally.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- [x] **Modular Clean Architecture**: Handled purely in tracking session generation (.NET) and iframe rendering (Next.js). No cross-domain violations.
- [x] **Provider Abstraction First**: The watermark is provider-agnostic. It runs over the iframe regardless of Telegram or YouTube.
- [x] **Security & Access Control by Default**: Name and phone are extracted securely from auth tokens and shielded by AES-256-GCM. Client cannot fake them.
- [x] **Academic Content Integrity**: Protects intellectual property by deterring screen recording.
- [x] **Single-Flow Registration**: Phone number relies on verified data from the registration flow.
- [x] **Observability**: Server errors generating the cipher log safely.
- [x] **Premium Editorial Design System**: Watermark uses subtle opacity and font shadows to remain visible but unobtrusive.
