**Feature Branch**: `026-whatsapp-dob-ux`
**Created**: 2026-03-31
**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 — تحقق حقيقي أوتوماتيك من واتساب عبر Evolution API (Priority: P1)

عندما يكتب الطالب رقم هاتفه في نموذج التسجيل، يتحقق النظام تلقائيًا وفعليًا من خوادم واتساب إذا كان الرقم مسجلاً على واتساب أم لا. النتيجة تظهر بجانب حقل الرقم بدون أي تفاعل من الطالب.

**المنظومة التقنية المتاحة**: يوجد Evolution API instance شغال على `evo.n8n-mazen.online` اسمه `Nader` — يوفر endpoint حقيقي `/chat/whatsappNumbers/{instance}` يتحقق من أي رقم فعليًا.

**Why this priority**: التحقق الحقيقي يضمن إمكانية التواصل مع الطالب عبر واتساب فعلاً. بخلاف مجرد التحقق من تنسيق الرقم، هذا يسأل خوادم واتساب مباشرةً.

**Independent Test**: اكتب رقم هاتف حقيقي عليه واتساب → يظهر "✓ الرقم على واتساب". اكتب رقم ليس عليه واتساب → يظهر "✗ الرقم ليس على واتساب".

**Acceptance Scenarios**:

1. **Given** الطالب أدخل رقم هاتف مصري صحيح (11 رقمًا), **When** يتوقف عن الكتابة لمدة ثانية أو ينتقل لحقل آخر, **Then** يظهر "جارٍ التحقق..." ثم النتيجة: "✓ الرقم على واتساب" أو "✗ الرقم ليس على واتساب"
2. **Given** الرقم المدخل ليس على واتساب, **When** يظهر المؤشر, **Then** تظهر رسالة تحذيرية واضحة لكن لا تمنع المتابعة
3. **Given** الرقم أقصر من 11 رقمًا, **When** لم يكتمل الرقم, **Then** لا يُرسل أي طلب ولا يُعرض مؤشر
4. **Given** فشل الاتصال بخدمة التحقق (Evolution API معطل), **When** تحدث مشكلة تقنية, **Then** يُعرض مؤشر محايد "تعذر التحقق" ولا يمنع الطالب من المتابعة

### User Story 2 — عرض السن ومعلومات الميلاد في لوحة البريفيو (Priority: P2)

بعد أن يختار الطالب تاريخ ميلاده، تُعرض معلومات السن الحسابية (سنه الحالي وعدد الأيام المتبقية على عيد ميلاده) في **لوحة البريفيو** على يمين الشاشة فقط، وليس بجانب حقل الإدخال نفسه.

**Why this priority**: عرض المعلومات في البريفيو يجعل منطقة الإدخال نظيفة ومركزة، بينما يعطي الطالب ملخصًا بصريًا جميلًا لبياناته.

**Independent Test**: اختر تاريخ ميلاد → لا شيء يظهر بجانب الحقل. انظر البريفيو (اليمين) → تظهر "سنك دلوقتي: X سنة" و"باقي X يوم 🎂".

**Acceptance Scenarios**:

1. **Given** الطالب اختار تاريخ ميلاد, **When** يتم عرض البريفيو, **Then** تظهر بطاقة السن + بطاقة الأيام في البريفيو فقط
2. **Given** تاريخ الميلاد لم يُختر, **When** يُنظر للبريفيو, **Then** تظهر بطاقة placeholder مطفأة
3. **Given** البطاقات في البريفيو, **Then** لا توجد أي معلومات حسابية بجانب حقل تاريخ الميلاد نفسه

### User Story 3 — تقييد التقويم على تواريخ قبل 2020 (Priority: P3)

عند فتح اختيار تاريخ الميلاد، يكون التقويم مقيدًا بحيث لا يمكن اختيار أي تاريخ بعد 31 ديسمبر 2019.

**Why this priority**: الطلاب على المنصة يجب أن يكونوا وُلدوا قبل 2020 ليكونوا في المراحل الدراسية المدعومة.

**Independent Test**: افتح التقويم → حاول اختيار 2020+ → disabled. اكتب يدويًا "2020-01-01" → رسالة خطأ.

**Acceptance Scenarios**:

1. **Given** الطالب يفتح التقويم, **Then** 2020+ غير قابل للاختيار
2. **Given** الطالب يكتب تاريخ يدويًا بعد 2019, **Then** خطأ تحقق واضح
3. **Given** الطالب يتصفح التقويم, **Then** آخر سنة متاحة هي 2019

---

### Edge Cases

- Evolution API بطيء أو معطل → مؤشر "تعذر التحقق" + يتابع بدون مشاكل
- الطالب غيّر الرقم بعد التحقق → يُعاد التحقق تلقائيًا
- تاريخ ميلاد مكتوب يدويًا بعد 2019 → Zod يرفض
- شاشة صغيرة بدون بريفيو → بطاقة السن تظهر في ملخص المراجعة

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: يجب أن يتحقق النظام فعليًا من واتساب عبر backend proxy يتصل بـ Evolution API
- **FR-002**: Backend يجب أن يوفر endpoint خاص لأن الـ API key لا يُكشف للـ frontend
- **FR-003**: يبدأ التحقق تلقائيًا عند اكتمال 11 رقمًا مصريًا بعد debounce 1 ثانية
- **FR-004**: يجب عرض حالة (جارٍ التحقق / ✓ على واتساب / ✗ ليس على واتساب / تعذر التحقق)
- **FR-005**: غياب واتساب تحذير إرشادي فقط — لا يمنع التسجيل
- **FR-006**: يجب عرض بطاقة السن والأيام في البريفيو فقط
- **FR-007**: يجب إزالة عرض السن من منطقة حقل تاريخ الميلاد
- **FR-008**: يجب تقييد تاريخ الميلاد على 2019-12-31 كأقصى تاريخ

### Key Entities

- **WhatsAppCheckResult**: نتيجة التحقق — `exists: boolean`, `jid: string`, `number: string`
- **PhoneVerificationState**: حالة الـ UI — `idle | checking | verified | not_found | error`
- **DateOfBirthConstraint**: الحد الأقصى `2019-12-31`

---

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: التحقق من واتساب يبدأ وينتهي في أقل من 5 ثوانٍ
- **SC-002**: 100% من الطلاب يرون نتيجة واتساب بدون ضغط أي زرار
- **SC-003**: معلومات السن تظهر فقط في البريفيو ولا تظهر بجانب الحقل
- **SC-004**: صفر طلاب يستطيعون التسجيل بتاريخ ميلاد 2020+

---

## Assumptions

- Evolution API instance `Nader` على `evo.n8n-mazen.online` متاح ومتصل
- الـ API Key يُحفظ في environment variables ولا يُكشف للـ frontend
- قيد 2019 يُطبَّق على تاريخ ميلاد الطالب فقط (ليس الوالدين)
- البريفيو موجود حاليًا في شاشة التسجيل (مُنفّذ في 025)
