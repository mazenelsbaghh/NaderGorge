# Data Model & Contracts: Exam and Question UI Enhancements

## Entities

### Question (Modifications)
Added attributes to the base (or extended) question model:
- `audioUrl` (string, nullable) - URL to the voice collection.
- `writtenCorrection` (string, nullable) - Rich text post-submission explanation.
- `hintText` (string, nullable) - Hint provided for the student.

### FindTheMistakeQuestion (New Subtype/Configuration of Question)
- `baseText` (string) - The complete context text containing the mistake.
- `mistakeStartIndex` (int) - The character start index of the correct mistake within the base text.
- `mistakeEndIndex` (int) - The character end index.
- `mistakeCorrectWord` (string) - (Optional) What the correct word should be, for correction display.

### EssaySubmission (Modifications/New Entity)
Represents the student's submission for an essay question.
- `id` (uuid)
- `studentId` (uuid)
- `questionId` (uuid)
- `answerText` (string) - Student's provided answer.
- `aiInitialScore` (float, nullable) - The grade assigned by the AI worker.
- `aiFeedback` (string, nullable) - The feedback generated by the AI worker.
- `teacherFinalScore` (float, nullable) - The final grade assigned/approved by the teacher.
- `teacherFeedback` (string, nullable) - Final or overriding feedback from the teacher.
- `status` (enum: PendingAI, RequiresTeacherReview, Graded) - The current state of the submission.

## State Transitions
**Essay Grading Lifecycle:**
1. Student submits -> `status` = PendingAI.
2. Background Job queued -> Gemini processes -> updates `aiInitialScore` and `aiFeedback` -> `status` = RequiresTeacherReview.
3. Teacher reviews, adjusts score/feedback if needed, and approves -> `status` = Graded.

## Interface Contracts (C# .NET API)

### `POST /api/admin/questions/{questionId}/audio`
Upload an audio file for a question.
**Input**: Multipart form data with audio file.
**Output**: 200 OK with `{ "audioUrl": "https://..." }`

### `POST /api/teacher/essay-submissions/{submissionId}/grade`
Teacher manually approves or overwrites the essay score.
**Input**:
```json
{
  "finalScore": 9.5,
  "feedback": "Great job, but missing one point."
}
```
**Output**: 200 OK
