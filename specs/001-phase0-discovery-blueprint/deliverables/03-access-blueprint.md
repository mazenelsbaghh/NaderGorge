# 03 — Access Blueprint (Code System)

## Code Types
Access to platform content is strictly governed by a pre-paid code system.
1. **Lesson Code:** Unlocks a specific, single lesson.
2. **Package Code:** Unlocks an entire package (e.g., a "Month" or "Chapter").
3. **Term Code:** A master code that unlocks multiple packages comprising an entire term.
4. **Promotional Code (Future scope):** Discount or trial codes for marketing.
5. **Referral Code (Future scope):** Codes generated by students for affiliate tracking.

## Code Behaviors
- **Single-use:** Every code can only be activated by one student account. Once activated, it is permanently bound to that account.
- **Batches:** Codes are generated in secure, trackable batches for distribution to centers or agents.
- **Content-Based Logic:** A code hard-links to specific content entities (Lesson IDs, Package IDs).
- **Duration-Based Logic:** A code may grant access for a fixed duration (e.g., 30 days) starting from the moment of activation.

## Activation Flow
1. **Entry:** Student enters the alphanumeric code in the portal.
2. **Validation:** System checks code validity, usage status, and expiration.
3. **Confirmation:** System displays what content the code will unlock.
4. **Selected Date (Optional future):** Student confirms activation date.
5. **Access Grant:** System creates a `StudentAccessGrant` record tying the student, the code, the content, and the duration.

## Stacking Rules
Students can activate multiple codes.
- **Lesson + Package:** If a student buys a Package Code that contains a Lesson they already unlocked via a Lesson Code, the broader Package Code supersedes the Lesson Code (duration is extended to the Package's duration).
- **Package + Term:** Similar to above, Term Code supersedes individual Package Codes.
- **Time Stacking:** If two identical Package Codes are entered (rare, but possible), their access durations do *not* append consecutively; they run concurrently from their respective activation dates.

## Expiration Rules
- **Pre-Usage Expiration:** Codes can have an absolute expiry date (e.g., "Valid until Dec 31st"). If not activated by then, the code becomes invalid.
- **Post-Activation Duration:** Once activated, the `StudentAccessGrant` has an absolute end date calculated based on the code's defined duration (e.g., +30 days from activation).

## Conflict Resolution
When overlapping content access is granted from multiple code types (e.g., student activated Lesson 1 code, then later activated Package 1 code):
- The platform always grants access based on the **most permissive** unexpired grant.
- As long as *any* valid, unexpired grant exists for a content ID for a specific student, access is allowed.
