# Data Model: Chapter Mindmap Generation

## Entities

### `TeacherPhoto`
Stores reference images used by the AI to understand the teacher's likeness for caricature generation.

- **Id** (`Guid`, PK)
- **TeacherId** (`String/Guid`, FK to User/Teacher)
- **FileUrl** (`String`): The path or URL of the uploaded image.
- **IsActive** (`Boolean`): Identifies if the photo is currently used for AI referencing.
- **UploadedAt** (`DateTime`)

### `LessonChapter` (Update Existing)
Adds a field to link the generated mind map image.

- **MindmapImageUrl** (`String`, nullable): The URL of the mind map generated by Gemini Image Generation API. Will be populated asynchronously by the worker.

### `AiJob` (Update Existing - Context)
If the AI processing is managed via a job system (BullMQ), we might need to update the AI job payload to signal that it should generate mind maps post-chaptering, and pass the reference image URLs down to the Node.js worker.

---

## Validation rules

- A teacher can upload multiple reference photos.
- The `MindmapImageUrl` must be a valid URL resolving to local storage (`/uploads/` or `.tmp/`). S3 is no longer used for this feature.
- During AI generation, if image creation fails, the chapter is still saved, but the `MindmapImageUrl` remains `null`.

## State transitions
- **Teacher Photo Upload**: Admin uploads `File` -> Stored locally (Local File System) -> DB record created -> Used actively in future AI Video Chapters Job invocations.
- **AI Processing**: Video Chapter Extracted -> Text Summarized -> Mind Map Image Generation Triggered via Gemini -> File Saved -> `LessonChapter.MindmapImageUrl` updated.
